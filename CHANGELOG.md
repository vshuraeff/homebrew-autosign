# Changelog

All notable changes to brew-autosign. Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/); versioning is [SemVer](https://semver.org/spec/v2.0.0.html).

## [0.2.0] — 2026-06-13

### Added

- **Configurable signing identity with auto-detection.** New `identity` subcommand (`show` / `list` / `set <name|sha1>` / `reset`) lets you sign with any codesigning identity in your login keychain — e.g. an Apple Development or Developer ID cert — instead of the managed self-signed default. Identities are auto-detected; the choice is stored in `~/.config/brew-autosign/identity`. `identity set` validates the identity, re-signs configured binaries with it, and reloads the agent. Nothing identity-specific is hardcoded.
- **`sign --force`** re-signs configured binaries even when already signed by another identity (used by `identity set` to apply a switch immediately). The automatic agent pass keeps its safe never-overwrite default.
- `status` now reports the active identity and its source (managed vs configured), and reads expiry for external certs from the keychain (30-day warning window; 365 days for the managed 10-year cert).

### Changed

- **More reliable triggering.** The LaunchAgent now also watches each prefix's long-lived `opt/` directory (retargeted on every `brew upgrade`), which survives the atomic delete+recreate of a per-package symlink that can orphan a vnode watch, and auto-covers configured-but-not-yet-installed packages without a manual `reload`. The periodic backstop tightened from 60 to 30 minutes.
- `sig_state` matches the signer by fixed-string comparison (not regex), so identities containing metacharacters — e.g. `Apple Development: name (TEAMID)` — are recognized correctly.

### Fixed

- Keychain ACLs no longer silently break when a second codesigning identity (e.g. a newly added Apple Developer cert) is present and a configured tool drifts between identities across upgrades. Standardize on one identity via `brew-autosign identity set`; see `docs/TROUBLESHOOTING.md`.

## [0.1.0] — 2026-05-17

Initial release.

### Added

- Single-file bash CLI `brew-autosign` with subcommands `setup`, `sign`, `reload`, `add`, `remove`, `list`, `status`, `uninstall`, `completions`, `help`. Per-subcommand `--help` for `add`, `remove`, `uninstall`, `completions`.
- Self-signed **ECDSA P-521** cert (SHA-512, 10-year validity) generated by `setup` and imported into the user's login keychain. Trusted for the user-domain `codeSign` policy. The strongest NIST curve Apple `codesign` accepts.
- LaunchAgent `dev.brew-autosign` auto-installed by `setup`. Triggers via `WatchPaths` on each configured package's `Cellar/<pkg>` directory, plus a one-hour `StartInterval` backstop against dropped or coalesced events.
- `packages.conf` driven, two forms per line: `<pkg>` (sign every unsigned executable in the keg's `bin/`) or `<pkg>:bin1,bin2,…` (specific binaries).
- Shell completions for **bash, zsh, fish** auto-installed by the Homebrew formula via `generate_completions_from_executable`, plus a `brew-autosign completions install|uninstall|generate <shell>|auto` manual path with marker-based idempotency.
- Release tooling: `scripts/bump.sh` driving `svu` (Conventional-Commits-based version detection) and `git-cliff` (CHANGELOG generation). Both pinned as dev deps in `Brewfile.dev`.
- GitHub Actions CI: `shellcheck` lint, macOS syntax + help smoke, input-validation rejection tests, `brew formula` parse + audit.

### Security

- **Provenance gate.** The agent refuses to sign anything inside a `Cellar/<pkg>/<ver>/` that lacks a brew-written `INSTALL_RECEIPT.json` (verified against minimum-size + required JSON keys). Defeats the "drop a fake binary into Cellar and wait for the agent to bless it" path.
- **No private key on disk.** After `security import` the `.key` and `.p12` are deleted from the private temp dir; only the public `codesign.crt` is kept for fingerprint / expiry display. An `EXIT` trap wipes key material on signal-driven termination too.
- **`uninstall --purge` deletes by SHA-1 fingerprint** (`security delete-{identity,certificate} -Z`), not by CN, so it cannot collide with another cert sharing the same Common Name. CN fallback only when no local cert file remains.
- **Plist XML escaped** (`& < > " '`) and `plutil -lint`-validated **before** the plist is moved into place. Path metacharacters cannot inject LaunchAgent keys.
- **Strict input validation.** Package names must match Homebrew formula naming (`^[a-z0-9][a-z0-9._+@-]{0,127}$`); binary names must be plain basenames (no slashes, traversal, control chars, whitespace). `add` validates at write time; `remove` matches by literal string. Invalid lines in `packages.conf` are rejected with a stderr warning and skipped.
- **`flock` resolved from fixed Homebrew prefixes only** (`$HOMEBREW_PREFIX/bin`, `/usr/local/bin`, `/opt/homebrew/bin`) — no `$PATH` fallback, so a malicious flock placed earlier in PATH cannot compromise the agent.
- `~/.local/share/brew-autosign` is `mode 700`.
- Threat model documented honestly in `docs/ARCHITECTURE.md`: the `-A` ACL on the private key (necessary for an unattended LaunchAgent) means a same-user attacker can still call `/usr/bin/codesign --sign brew-autosign-local` directly. Provenance gate reduces but does not eliminate this. Bounded to user-domain `codeSign` policy; no system trust, no Gatekeeper bypass, no notarization.

### Correctness

- **`sig_state` distinguishes four states**: signed-by-us / unsigned / signed-by-other / codesign-error. Opaque codesign errors are surfaced, never silently confused with "signed by other".
- **Signing failures propagate.** `cmd_sign` returns non-zero if any binary failed.
- **Atomic config writes.** `add`/`remove` acquire a dedicated `.config.lock` for the entire read-modify-write-reload cycle; the temp file lives next to `$CONFIG` so `mv` is a same-filesystem rename.
- **Clock-skew-safe debounce.** `.last_run` future timestamps (Time Machine restore, clock changes) are clamped to zero instead of permanently suppressing sign passes.
- **Stability check** (size + mtime stable across two reads, up to 10 × 200 ms) replaces a fixed `sleep` before signing, avoiding half-written binaries mid-`brew upgrade`.
- **Identity preflight** logs a precise diagnostic when the keychain identity is unavailable, instead of failing silently.
- **macOS-only assertion** at startup; FAT64 Mach-O magic (`bfbafeca`, `cafebabf`) recognised in addition to thin and classic universal binaries.
- **Bash 3.2 compatible.** The LaunchAgent invokes `/bin/bash`, which on macOS is 3.2.57 — no `mapfile`, no `local -n`, etc.
- **`cert_days_left` forces `LC_ALL=C`** around openssl/date so non-English locales don't break expiry parsing.
- `find-certificate` export refuses multi-cert PEM bundles with a clear diagnostic rather than silently picking one.

### UX

- **`status`** leads with a `status: ok | degraded | not-set-up` summary line. Reports identity presence, cert fingerprint and days-until-expiry, LaunchAgent state, watched paths, configured-but-not-installed packages, and recent stderr from the agent. Idle on-demand LaunchAgent state is shown as `idle (waiting for triggers)` rather than the misleading `not running`.
- **`reload`** warns about configured-but-not-installed packages (no watch path emitted).
- **`add`** warns when the package isn't installed yet.
- **`setup`** is idempotent and resumable; re-running after the trust prompt was dismissed re-attempts only the unfinished steps and verifies the trust round-trip.
- **`uninstall --purge`** removes cert + Keychain identity + data dir with a typed confirmation prompt.
- **`list`** has a `STATE` column with normalized casing and a provenance marker (`brew` / `nokeg`).
- **Expiry warning at 365 days** in `status`.
- Section headers in CLI output use terminfo `tput bold` + `─` underline; ANSI codes are emitted only when stdout is a tty.

### Known limitations

- Same-user attacker with code execution can still call `/usr/bin/codesign --sign brew-autosign-local` directly (the unattended-LaunchAgent compromise). See `docs/ARCHITECTURE.md`.
- Shared `Cellar` across multiple macOS user accounts is not supported: each user's keychain holds a different identity, so signatures appear as "signed-by-other" between users.
- iCloud Keychain does not sync local code-signing identities; each Mac runs `brew-autosign setup` independently.
- Only `bin/<…>` binaries are signed; `sbin/` / `libexec/` are not covered in v0.1.
- Cert lifetime is 10 years; rotation forces the one-time "Always Allow" cycle to re-bind ACLs.
- Pre-existing Keychain items created before install have ACLs anchored to the previous designated requirement. The first post-install access prompts "Always Allow" once per item. After that, all future `brew upgrade`s are silent.
